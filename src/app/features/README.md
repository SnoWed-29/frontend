# Features Module

This folder contains feature-specific modules and pages. Each feature represents a major section of the application.

## Structure

- **auth/** - Authentication pages (login, register, etc.)
- **internships/** - Internship management pages
- **students/** - Student management pages
- **teachers/** - Teacher management pages
- **reports/** - Report pages

## Guidelines

- Each feature should be self-contained
- Feature components should use shared components from the shared folder
- Feature services should use core services
- Each feature can have its own routing module
- Keep feature-specific logic within the feature folder
